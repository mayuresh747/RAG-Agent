# Contributing to RAG Agent

We welcome contributions! Please follow these guidelines to ensure a smooth collaboration process.

## Getting Started

1.  Fork the repository.
2.  Create a feature branch: `git checkout -b feature/my-new-feature`.
3.  Commit your changes: `git commit -am 'Add some feature'`.
4.  Push to the branch: `git push origin feature/my-new-feature`.
5.  Submit a pull request.

## Code Style

- Follow PEP 8 guidelines for Python code.
- Ensure all new features are accompanied by unit tests in the `tests/` directory.
- Place new source files in `src/` organized by module.

## Reporting Issues

If you find a bug or have a feature request, please open an issue with detailed steps to reproduce or a clear description of the desired functionality.
