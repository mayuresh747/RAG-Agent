Why This Matters

The Policy Document Problem

Government agencies produce thousands of interconnected policy documents. They're critical for compliance — and nearly impossible for AI to navigate accurately.

Linguistic Complexity

Nested conditional clauses, cross-references between statutes, temporal qualifiers, and exception chains make every sentence a puzzle.

Structural Complexity

4-8 levels of nesting: Title → Chapter → Section → Subsection. Answering a question may require traversing multiple levels.

Semantic Density

A single paragraph may define eligibility criteria, specify exceptions, set thresholds, and reference other documents simultaneously.

Inter-Document Dependencies

RCW mandates → WAC implements → SMC localizes → DIR Rules interpret. A web of hierarchical authority that AI must understand.

What We're Investigating

Research Questions

Six fundamental questions that guide every experimental phase.

RQ1 Baseline Knowledge Assessment
  • How does accuracy vary across different policy sources (RCW, WAC, SMC, DIR, IBC, SPU)?
  • What types of errors does the bare LLM make (hallucination, outdated info, sycophancy)?
  • Does the LLM demonstrate awareness of its own knowledge limitations?
RQ2 System Prompt Impact
  • What is the marginal improvement from a basic vs. engineered system prompt?
  • Can prompt engineering alone reduce hallucination rates?
  • How does prompt complexity affect response latency and token usage?
RQ3 RAG Component Attribution
  • Which component has the largest single impact on faithfulness?
  • Which component has the largest single impact on answer relevance?
  • Are there interaction effects between components?
RQ4 Scalability Analysis
  • Is there a "saturation point" beyond which adding documents degrades performance?
  • How does retrieval latency scale with corpus size?
RQ5 Query Complexity Response
  • At which phases do complex queries begin to be handled effectively?
  • What is the relationship between query complexity and required retrieval depth?
  • Do adversarial queries expose systematic vulnerabilities?
RQ6 Advanced Techniques Impact
  • Does hybrid search outperform pure vector search for policy content?
  • How much does reranking improve final answer quality vs. its computational cost?
  • Are guardrails necessary, and do they reduce useful responses?

Phase 0: Bare LLM Baseline

Variable: No context, no system prompt, no retrieval

Establish the absolute baseline by testing GPT-5.1's ability to answer policy questions using only its pre-trained knowledge. No system prompt, no retrieval, no additional context.

Completed

Phase 1: Basic System Prompt

Variable: Addition of basic system prompt

Add a simple system prompt that identifies the agent's role and jurisdiction scope. Measures whether basic instruction-following reduces hallucination and improves accuracy.

Completed

Phase 2: Engineered System Prompt

Variable: System prompt engineering sophistication

Replace the basic prompt with a carefully engineered "Regulatory Auditor" persona prompt. Tests whether detailed prompt engineering alone can meaningfully improve response quality.

Current Phase

Phase 3: Basic RAG Pipeline

Variable: Addition of basic RAG pipeline

Introduce the core retrieval pipeline: embed queries with text-embedding-3-large, search ChromaDB, inject retrieved chunks into the prompt. The first phase where the LLM has access to actual policy documents.

Upcoming

Phase 4: Chunking & Embedding Strategies

Variable: Chunking and embedding strategies

Compare chunking approaches (fixed-size, recursive, legal-aware separators) and embedding models. Optimize how documents are split and represented in vector space.

Upcoming

Phase 5: Hybrid Search & Reranking

Variable: Retrieval strategy (hybrid search, reranking)

Layer BM25 keyword search with vector search, and add cross-encoder reranking (Cohere Rerank v3, BGE Reranker v2) to improve retrieval precision.

Upcoming

Phase 6: Multi-Source Corpus Scale

Variable: Corpus scale (~all 6000+ documents from 8 sources)

Scale from a subset to the full corpus across all 8 WA State/Seattle sources. Tests how retrieval quality holds up at scale with potential collection confusion.

Upcoming

Phase 7: Query Decomposition

Variable: Query processing (decomposition, multi-hop chains)

Break complex queries into sub-queries, execute multiple retrieval cycles, and synthesize results. Targets Level 3-5 query complexity.

Upcoming

Phase 8: Guardrails & Citation Enforcement

Variable: Guardrails and citation enforcement

Add domain restrictions, hallucination detection, citation requirements, and confidence scoring. Tests whether guardrails improve reliability without sacrificing coverage.

Upcoming

Phase 9: Agentic Architecture

Variable: Agentic architecture with tool use

Upgrade from a single-turn RAG to an agent that can plan retrieval strategies, use tools, iterate on queries, and autonomously decide when it has enough information to answer.

Evaluation Framework

Query Taxonomy

Six levels of complexity, from simple factual lookups to adversarial edge cases — designed to stress-test every aspect of the system.

Level 1: Simple Factual Queries

Queries answerable from a single fact in a single document. The answer is explicit and unambiguous.

SubtypeExample Query
Definition LookupWhat is the definition of a "large vehicle" parking space?
Threshold / LimitWhat is the maximum permitted slope for a driveway?
Yes/No FactualAre you allowed to park a vehicle on a planting strip?
Process / ProcedureWhat happens to temporary parking facilities after construction is finished?

Design Principles

  • Answer findable in one chunk regardless of chunking strategy
  • Expected difficulty: Low — serves as a sanity check

Level 2: Multi-Fact Single-Source Queries

Queries requiring multiple facts from the same document, potentially from different sections.

SubtypeExample Query
Comprehensive RequirementWhat are all conditions to establish parking accessory to a single-family structure?
Scope and BoundaryIn multifamily zones, where is surface parking permitted and what areas are excluded?
Conditional / ExceptionUnder what exceptions is vehicular access permitted through a required yard?

Design Principles

  • Answer requires 2-4 chunks from the same document
  • Expected difficulty: Medium

Level 3: Multi-Document Reasoning

Queries requiring information from multiple documents, potentially from different sources (RCW, WAC, SMC, etc.).

SubtypeExample Query
Cross-ReferenceHow does RCW 36.70A.681's prohibition on ADU parking connect to Seattle's SMC 23.53.003?
ComparativeHow does WA State's parking standard for middle housing compare to Seattle's obligations?
Compliance SynthesisWhat overlapping state and local regulations govern off-street parking for a new fourplex?

Design Principles

  • Answer requires chunks from 2+ different documents
  • May require resolving conflicts between documents
  • Expected difficulty: High

Level 4: Multi-Hop Reasoning

Queries requiring chains of reasoning — the answer to question A provides context for question B.

SubtypeExample Query
Chain-of-ReferenceIf a development in the Downtown Urban Center is exempt from design review, what parking standards apply?
Conditional ChainUnder WAC 365-196-875, what are the two conditions limiting off-street parking requirements?
AggregationWhat are the exact minimum dimensions for small, medium, and large vehicle parking spaces?

Design Principles

  • Answer requires 3+ retrieval-reasoning cycles
  • Output of one retrieval informs the next query
  • Expected difficulty: High

Level 5: Temporal & Versioning

Queries about how policies have changed over time, which version is current, or effective dates.

SubtypeExample Query
Version HistoryAre existing parking spaces required to be modified if they don't meet new state requirements?
Effective DateWhat is the cutoff date for temporary surface parking to be eligible for permit renewal?
SupersessionWhat happens if a local city's regulations conflict with state requirements in RCW 36.70A.698?

Design Principles

  • Requires understanding temporal metadata
  • Must identify the most current version among multiple documents
  • Expected difficulty: High

Level 6: Adversarial & Edge-Case

Queries designed to test system robustness — out-of-scope topics, trick questions, ambiguous policy areas.

SubtypeExpected Behavior
Out-of-ScopeSystem should acknowledge it cannot provide statewide data beyond its corpus
Trick / MisleadingSystem should correct false premises (e.g., "Seattle has no minimum parking sizes" is wrong)
AmbiguousSystem should note that the answer genuinely depends on interpretation
Sensitive / ControversialSystem should present facts and decline to make normative legal conclusions

Design Principles

  • Tests system boundaries, not just capabilities
  • Evaluates graceful degradation and appropriate refusal
  • Expected difficulty: Varies (tests a different dimension than knowledge)

Phase 0 Results

How well does a bare LLM answer policy questions with zero context? Not well.

The bare LLM consistently ignores localized Seattle/WA State facts, relying on generalized knowledge of U.S. zoning and planning laws. Multi-hop reasoning queries (L4) show a 90% hallucination rate — the model confidently fabricates plausible but incorrect answers. This establishes the baseline that RAG augmentation must improve upon.
